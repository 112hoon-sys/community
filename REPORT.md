# K-Connect 커뮤니티 기능 구축 보고서

## 1. 배경
- 외국인 커뮤니티 사용자들의 생활/구인/중고 거래 니즈가 증가하면서, 한국어 장벽과 정보 접근성이 문제로 작용.
- 기존 커뮤니티는 게시판 중심으로만 운영되어 실시간 소통과 거래 연계가 약함.
- 알림/채팅 부재로 인해 게시글 반응 속도가 느리고, 사용자 참여 지속성이 낮음.

## 2. 목적
- 다국어 커뮤니티 사용자들이 생활/구인/중고 거래를 빠르게 연결할 수 있는 플랫폼 제공.
- 게시판 + 알림 + 1:1 채팅 + 푸시로 실사용 흐름을 완성.

## 3. 목표
1. 홈 화면에서 마켓/일자리 추천 게시글 노출
2. 게시글 상세에서 1:1 채팅 시작 및 다중 스레드 관리
3. 댓글/좋아요/채팅 알림 저장 및 알림함 연동
4. 웹 푸시 알림으로 반응 속도 개선
5. 모바일/PC 반응형 UI 일관성 확보
6. 언어 선택 UI 간소화(국기 기반)

## 4. 계획
- 1단계: UI/레이아웃 개선
  - 홈 구성 정리, 카드 2열, 배너 광고화
  - 언어 선택 국기 UI 적용
- 2단계: 게시판 + 채팅 연동
  - 마켓/구인 상세에서 채팅 생성
  - 글쓴이는 여러 채팅 선택 UI 제공
- 3단계: 알림 기능 확장
  - 댓글/좋아요/채팅 알림 저장
  - 알림함 UI 연동
- 4단계: 웹 푸시 적용
  - VAPID 기반 푸시 구독
  - 알림 발생 시 자동 푸시 발송
- 5단계: 테스트 및 안정화
  - 게시글/채팅/푸시 플로우 점검
  - 이미지 업로드 및 리소스 접근성 확인

## 5. 구현 내용 요약
### 5.1 UI/레이아웃
- 로그인 사용자 정보 및 로그아웃 버튼 표시
- 반응형 레이아웃 재정비
- 마켓/구인 카드 PC 2열 배치
- 광고 배너 고급형 디자인 적용
- 언어 선택을 국기 UI로 변경

### 5.2 게시판/채팅
- 마켓·구인 상세에서 1:1 채팅 시작
- 작성자는 여러 채팅 스레드를 선택 가능
- 채팅 메시지/읽음 처리/미읽음 카운트 표시

### 5.3 알림/푸시
- 댓글/좋아요/채팅 발생 시 알림 저장
- 알림함 UI에서 미읽음 카운트 표시
- 웹 푸시 구독/발송 로직 구축(VAPID)

## 6. 예측 효과
- 채팅/알림/푸시 연동으로 게시글 반응 속도 개선
- 거래 전환율 상승(마켓/구인 연결)
- 사용자 재방문율 증가(알림 기반)
- 다국어 사용자 접근성 개선

## 7. 현재 상태
- DB 스키마 반영 완료
- 푸시 VAPID 키 발급 및 환경변수 등록 완료
- 서버/프론트 재시작 후 테스트 가능
